# Audience Calibration

## Decision

Choose the teaching depth, language, pacing, and scaffolding for the audience before drafting the artifact.

## Signals/constraints

- The learner's prior knowledge is unclear or mixed.
- The delivery time is short relative to the topic.
- The user needs a tone that is encouraging, professional, or workshop-friendly.
- The topic includes programming or architecture abstractions that can overload beginners.

## Options

- Beginner: use concrete scenarios, fewer terms, more worked examples, and more retrieval checks.
- Intermediate: assume basic vocabulary, focus on tradeoffs, contrast cases, and guided practice.
- Advanced: move faster through basics, emphasize edge cases, decision criteria, and transfer.
- Mixed audience: layer explanations with a simple main path and optional deeper notes.

## Recommendation rule


Default to mixed-audience scaffolding when the audience is not specified: a simple main path, optional depth, and low-pressure checks.

## Tradeoffs

- More scaffolding improves access but reduces topic coverage.
- Faster pacing respects experienced learners but can hide prerequisite gaps.
- Mixed-audience layering costs structure time but reduces exclusion.

## Verification

- State the chosen audience assumption.
- Ensure learning goals describe observable learner actions.
- Check that the first example does not require unstated prerequisite knowledge.
