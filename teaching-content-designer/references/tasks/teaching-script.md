# Teaching Script

## When to use

Use when the user wants a speakable script, transcript, narration, demo talk track, or presenter notes for teaching or sharing.

## Inputs

- Audience, delivery length, and desired tone
- Lesson outline or source material
- Required examples, demo moments, or architecture/code concepts
- Interaction style, such as questions, pauses, audience prediction, or hands-on practice

## Steps

1. Identify the main learner transformation: what the learner can do after the script.
2. Draft in a speakable voice with short paragraphs, transitions, pauses, and learner-facing questions.
3. Use worked example fading: explain one complete case, invite prediction on a variation, then ask learners to apply.
4. Add planned interaction points with expected learner thinking.
5. Add misconception repair wording where the topic often causes confusion.

## Safety gates

- Do not write a monologue that only dumps definitions.
- Do not over-script learner responses as if they must answer perfectly.
- Do not make unsupported technical claims; delegate or mark them for verification.

## Outputs

- A speaker-ready script
- Presenter cues for pauses, questions, and transitions
- Expected learner reactions or checks
- Misconception repair language
- Closing recap and next action

## Verification

- Read the script for speakability: sentences should work aloud.
- Check that the script alternates explanation with learner thinking.
- Confirm the ending asks learners to retrieve, apply, or review.
