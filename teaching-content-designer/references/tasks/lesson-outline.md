# Lesson Outline

## When to use

Use when the user wants a lesson, course segment, workshop block, talk outline, syllabus slice, or teaching flow before writing full prose.

## Inputs

- Audience level, prior knowledge, and likely motivation
- Topic, learning goal, and delivery time
- Format constraints such as live class, internal sharing, workshop, video, or article
- Required technical scope and any source material

## Steps

1. Calibrate the audience with `audience-calibration` when level or time is unclear.
2. Load `references/pedagogy.md` and choose the smallest concept ladder that reaches the goal.
3. Break the lesson into motivation, prerequisite activation, core concept, example, practice, retrieval, and wrap-up.
4. Add interaction points and expected learner responses for live delivery.
5. Mark technical claims that need domain-skill verification.

## Safety gates

- Do not cram a full curriculum into one outline.
- Do not list topics without showing why the order helps learning.
- Do not assume learners can transfer from explanation to practice without a guided step.

## Outputs

- A time-aware lesson outline
- Observable learning goals
- Concept progression
- Interaction and retrieval points
- Follow-up practice or spaced review prompts

## Verification

- Confirm each segment supports one learning goal.
- Check that every new concept has either an example, an interaction, or a practice moment.
- Note any technical detail that should be verified by a domain skill.
