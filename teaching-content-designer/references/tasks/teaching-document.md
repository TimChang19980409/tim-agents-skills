# Teaching Document

## When to use

Use when the user wants a learner-facing handout, teaching note, tutorial document, article, onboarding guide, or written explanation.

## Inputs

- Target reader and prior knowledge
- Topic, scope, and source material
- Intended use: pre-read, follow-along, post-class notes, self-study, or reference
- Required examples, diagrams, exercises, or checkpoints

## Steps

1. Define the reader's starting point and the action they should be able to take after reading.
2. Start with motivation and a concrete scenario before formal vocabulary.
3. Sequence concepts with headings that reflect learner questions, not only system structure.
4. Add examples, checks for understanding, and practice prompts.
5. Add a compact recap and spaced review prompts when retention matters.

## Safety gates

- Do not turn the document into a reference manual unless requested.
- Do not hide prerequisite knowledge inside advanced examples.
- Do not use jargon before giving the learner a usable mental model.

## Outputs

- A structured teaching document
- Observable learning goals
- Concept ladder and examples
- Checks for understanding
- Common misconceptions and repair notes
- Recap and next practice

## Verification

- Ensure a learner can identify what to do, why it matters, and how to practice.
- Check that every section either motivates, explains, demonstrates, asks, or reviews.
- Mark any technical statement that should be verified against current docs.
