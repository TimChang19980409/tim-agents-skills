# Pedagogy

Use these principles when designing teaching artifacts for education, programming, or architecture topics.

## Learning principles

- Cognitive load: introduce a small number of new ideas at a time; separate essential difficulty from incidental notation, tooling, or terminology.
- Prior knowledge activation: start from what the audience already understands, then bridge to the new abstraction.
- Worked example fading: show a complete example first, then a partially guided variation, then independent learner action.
- Retrieval practice: ask learners to recall or explain before showing the answer; prefer reasoning prompts over trivia.
- Spacing: add a short D+1, D+7, or later review prompt when the artifact is meant to support lasting learning.
- Interleaving: mix explain, read, predict, debug, compare, refactor, or design tasks instead of repeating one task shape.
- Active learning: include moments where learners predict, choose, explain, apply, or critique.
- Misconception repair: name likely wrong mental models and replace them with a better one, using a concrete contrast.
- Motivation: connect the topic to a real job-to-be-done, pain point, or decision the learner recognizes.
- Psychological safety: make confusion normal, use invitations instead of blame, and avoid making the learner perform without support.

## Artifact checklist

- State who the learner is and what they can already do.
- Define learning goals as observable actions.
- Build a concept ladder with no hidden jumps.
- Mark interaction points and the expected learner action.
- Add retrieval checks before answers.
- Add one transfer exercise or real-world variation.
- Include wording for common confusion.
- Close with recap and next practice.

## Style rules

- Use concrete examples before abstract definitions when the concept is unfamiliar.
- Prefer "先觀察現象 -> 再命名概念 -> 最後使用規則" for programming and architecture topics.
- Avoid long lists of facts unless each item is connected to a learner decision.
- Avoid novelty overload: do not introduce new syntax, framework behavior, and architecture vocabulary in the same step unless the audience is advanced.
