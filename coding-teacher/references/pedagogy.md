# Pedagogy (Applied Learning Science)

Use these principles to make learning stick, transfer, and become usable in real projects:

## Core principles (must show up in every step)

1. **Retrieval practice**
   - Include a short “Retrieval Quiz” at the end of each step.
   - The quiz must be answerable without looking at notes; prefer explanations and reasoning over trivia.

2. **Spacing**
   - Attach a “Spaced Review Card” with prompts for D+1 / D+7 / D+30.
   - When the user returns, start by asking 2–3 retrieval questions before moving on.

3. **Interleaving**
   - Mix task types across steps: implement feature, read code, debug, refactor, test, document.
   - Avoid doing 5 steps in a row of the exact same task pattern.

4. **Cognitive load control**
   - Keep each step’s new concepts small.
   - Use worked example → partially guided → independent (fading).

5. **High-quality feedback**
   - Feedback must be actionable: “what”, “why”, “how to verify”.
   - Prefer the highest ROI fix first; do not overwhelm with a long list.

6. **Desirable difficulties**
   - Once the user can pass the MVP, add small variations/constraints to force transfer.
   - Examples: new input shape, extra edge case, performance constraint, different environment.

## How to embed these into the step template

Every step must include:

- Timebox (Minimum/Typical/Stretch)
- Deliverables Contract + Verification
- 2+ exercises (basic + advanced), ideally interleaved task types
- Retrieval Quiz
- Spaced Review Card (D+1/D+7/D+30)
