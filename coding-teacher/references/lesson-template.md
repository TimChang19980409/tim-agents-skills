# Lesson Step Template

Use this template for every step you teach.

## Step header

- **Step ID**: `Sxx` (or split `Sxx-a`, `Sxx-b`, …)
- **Title**
- **Difficulty**: 1–5
- **Timebox**
  - **Minimum**: _N_ minutes (MVP only)
  - **Typical**: _N_ minutes (core deliverables)
  - **Stretch**: _N_ minutes (quality extras)

## Objective (measurable)

Define what “done” means in 1–3 bullet points.

## Prerequisites (minimal)

Only list what is required for this step.

## Design thinking (Why / Tradeoffs / Alternatives)

- Why this approach
- Key tradeoffs
- Alternatives and when you’d choose them

## Deliverables Contract (auto-scannable)

Define the submission folder structure. This must be strict enough to scan.

- **Required**
  - List required files/folders and brief purpose
- **Allowed**
  - Allowed extra files (logs, notes, etc.)
- **Forbidden**
  - Secrets (`.env` with tokens), large binaries, build artifacts, vendor folders, etc.

## Verification (do not run without confirmation)

Provide commands the user can run to verify locally. If there are multiple ecosystems, provide the most likely one and alternatives.

## Submission Contract (the user’s homework message)

The user submits only:

- `Step ID`: `Sxx`
- `Submission path`: path to the **step folder** (not repo root)

Notes:

- If the user is working inside a larger custom repo, the step folder must be a subfolder (do not submit the repo root).

## Complete code (only when you provide starter code)

When you provide code, include:

- File tree
- Full contents of each file
- How to run / test / verify

## Learning points

List 3–7 takeaways.

## Exercises

At least two:

- Basic exercise (small variation)
- Advanced exercise (adds a desirable difficulty)

## Common mistakes & debugging hints

List the top pitfalls for this step and how to diagnose them.

## Retrieval Quiz (no notes)

2–5 questions.

## Spaced Review Card

- **D+1**: prompts/questions
- **D+7**: prompts/questions
- **D+30**: prompts/questions
