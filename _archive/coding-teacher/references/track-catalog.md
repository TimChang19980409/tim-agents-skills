# Track Catalog (Any language, any domain)

Use this catalog to choose a learning track based on the user's goal/output. If the user is unsure, ask what they want to build (a concrete artifact) and choose the shortest path to an MVP.

## Track selection rule

1. Ask for the target artifact (e.g., “a REST API”, “a mobile app”, “a firmware driver”, “a CLI script”).
2. Choose the closest track below.
3. Produce a minimal syllabus with 3 milestones and 6–12 steps.

## Tracks (8)

1. **Web Frontend** (React/TypeScript or equivalent)
2. **Web Backend** (Node/Spring/Go/etc.)
3. **Mobile** (React Native/Flutter/etc.)
4. **Embedded** (C/C++/RTOS)
5. **Systems** (C/Rust/OS fundamentals)
6. **Scripting** (Python automation)
7. **Data/ML** (Python + data workflows)
8. **DevOps** (Docker/CI/CD)

## Minimal milestone pattern (applies to all tracks)

- **M1**: runnable “hello product” (MVP skeleton)
- **M2**: core feature + correctness + edge cases
- **M3**: quality hardening (tests, refactor, security/perf as relevant)
