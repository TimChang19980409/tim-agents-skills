# Source Priority

## Decision

Choose which sources should dominate the browsing workflow.

## Signals/constraints

- Technical or high-stakes guidance
- Need for official docs versus broad market coverage
- User asking for community perspectives or installable skills

## Options

- Official or primary docs first
- News or broad web sources for market/current-event coverage
- Archive extensions for doc-specific or skill-discovery flows

## Recommendation rule

Use official or primary sources first for technical, medical, legal, or financial guidance. Use broader web or news sources only when the question is inherently comparative or topical.

## Tradeoffs

- Official docs maximize authority but may miss practitioner nuance
- Broader sources add coverage but raise noise and reliability risk
